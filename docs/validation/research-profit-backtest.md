# Research Profit Backtest Lane

## Purpose

This lane performs an offline, research-only portfolio simulation on local `price_features_and_forward_labels` artifacts.

It is designed to answer a narrow descriptive question: did simple feature-driven, long-only baskets produce portfolio-level profit evidence after costs and drawdowns?

## Scope

- Input: local price-feature/label artifact JSON generated by `research:price-features`.
- Output: JSON report under `.dexter/signal-engine/research/analysis/`.
- Data source: existing local artifact only.
- Execution: next available trading day after the signal date.
- Price basis: `adjustedClose` when available, otherwise `close`.
- Costs: fixed bps applied on entry and exit.

Out of scope:

- live trading
- auto-trading
- model training
- production policy tuning
- production signal logic changes
- `runDailyScan` behavior changes
- backtest/walk-forward validation changes
- live paid provider calls

## Strategy Families

V1 includes deterministic, config-driven long-only strategies:

- `drawdown_reversal_20d`: rank `drawdown_252d` ascending, buy deepest drawdown names, hold 20 trading days, weekly rebalance.
- `ret_1d_reversal_5d`: rank `ret_1d` ascending, buy worst 1-day return names, hold 5 trading days, weekly rebalance by default.
- `sma20_gap_reversion_20d`: rank `sma_20_gap` ascending, buy most negative SMA20 gap names, hold 20 trading days, weekly rebalance.

V1 uses non-overlapping baskets. If a basket is open, later rebalance signals are skipped until that basket exits.

## Baselines

Every report must compare strategies against:

- `equal_weight_buy_hold` over the same artifact universe and date range.
- `cash`.
- `SPY_buy_hold` only when `SPY` exists in the artifact.

The equal-weight buy-and-hold baseline is the primary benchmark. Strategy results should not be interpreted without this comparison.

## Metrics

Each strategy and baseline reports:

- total return
- CAGR
- annualized volatility
- Sharpe with 0 risk-free rate
- max drawdown
- Calmar
- win rate
- average and median trade return
- number of trades
- turnover
- average exposure
- best and worst month
- benchmark-relative return
- benchmark-relative max drawdown

## Profit Verdicts

V1 verdicts are descriptive gates:

- `reject`: does not beat buy-and-hold return and has drawdown at least as bad as buy-and-hold.
- `weak`: beats buy-and-hold return but has worse drawdown or weak Sharpe.
- `research_candidate`: beats buy-and-hold return, has acceptable drawdown, better Sharpe, and enough trades.
- `expand_universe`: interesting result but the trade count or sample breadth is too small.

These verdicts are not trading recommendations and are not permission to train or tune production behavior.

## CLI Usage

```bash
npm run research:profit-backtest -- \
  --in .dexter/signal-engine/research/price-features/price-features-first-multiticker-2026-04-26.json
```

Optional flags:

- `--out <path>`
- `--initial-capital <number>`
- `--cost-bps <number>`
- `--top-n <number>`
- `--max-positions <number>`
- `--min-trades <number>`
- `--ret-1d-rebalance daily|weekly`
- `--json`

## Interpretation Guardrails

Profit metrics are historical simulation only. They can be useful for research triage, but they are not evidence of live-trading performance.

Do not train a model, tune production policy, or change production signals unless profit evidence survives a broader universe, separate holdout periods, and review against equal-weight buy-and-hold.
